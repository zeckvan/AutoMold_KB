# 案例6：急單搶佔排程 → 優先處理

## 前置條件
- 存在一般工單與急單，且同時有型體 Size 欠數。
- 急單標記（Priority = High 或 RushOrder = Y）。
- AutoMold_ArticleMold.SumBodySizeNeed > 0。

## 測試步驟
1. 系統讀取工單列表，發現急單存在。
2. 檢查急單是否符合排產條件（有欠數、尚未完成排模）。
3. 系統優先將急單排入 AutoMold_Temp。
4. 更新 AutoMold_ArticleMold 欠數與上模數。
5. 更新 AutoMold_WorkMold 已排與未排數。

## 預期結果
- 急單工單先於一般工單分派完成。
- AutoMold_Temp 先產生急單紀錄，再處理其他工單。
- 排模結果符合「急單優先」規則。
